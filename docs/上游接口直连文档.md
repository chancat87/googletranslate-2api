# Google Translate 上游接口直连文档

> 从 `app/providers/googletranslate_provider.py` 抓取的真实参数。供直连调试、移植、对接其他客户端参考。
>
> 来源：`stackoverflow.ai` 抓包。Key 通过浏览器 DevTools 的 `x-goog-api-key` 获取。

---

## 1. 端点

```
POST https://translate-pa.googleapis.com/v1/translateHtml
```

仅此一个端点。无 REST 路径参数、无 query string。

---

## 2. 认证

请求头携带 API Key，**不是** query 参数：

```
x-goog-api-key: <你的 GOOGLE_API_KEY>
```

Key 获取方式：
1. 打开 `https://stackoverflow.ai/`
2. DevTools → Network → 触发一次翻译
3. 找 `translateHtml` 请求 → Request Headers → 复制 `x-goog-api-key` 值

Key 会失效，需定期重抓。

---

## 3. 请求头

完整必须头（少任一项可能被拒）：

| Header | 值 | 说明 |
|--------|-----|------|
| `Content-Type` | `application/json+protobuf` | 注意是 `+protobuf`，不是纯 `application/json` |
| `x-goog-api-key` | `<KEY>` | 见 §2 |
| `Origin` | `https://stackoverflow.ai` | 模拟来源 |
| `Referer` | `https://stackoverflow.ai/` | 模拟来源 |
| `User-Agent` | Chrome UA | 任意现代浏览器 UA |
| `Accept` | `*/*` | |
| `Accept-Language` | `zh-CN,zh;q=0.9,en;q=0.8` | 可选 |

---

## 4. 请求体

**不是标准 JSON 对象**，是嵌套数组（protobuf 风格）：

```json
[
  [[ "<待翻译文本>" ], "<source_lang>", "<target_lang>"],
  "te_lib"
]
```

字段说明：

| 位置 | 字段 | 说明 |
|------|------|------|
| `[0][0][0]` | text | 待翻译文本，字符串 |
| `[0][1]` | source_lang | 源语言码，`"auto"` 自动检测 |
| `[0][2]` | target_lang | 目标语言码 |
| `[1]` | 固定 | `"te_lib"`，翻译引擎标识 |

语言码遵循 ISO 639-1 / Google 惯例：`zh-CN`、`en`、`ja`、`ko`、`fr`、`de` 等。完整列表见 `app/core/languages.py`。

### 示例：英译中

```bash
curl -X POST "https://translate-pa.googleapis.com/v1/translateHtml" \
  -H "Content-Type: application/json+protobuf" \
  -H "x-goog-api-key: $KEY" \
  -H "Origin: https://stackoverflow.ai" \
  -H "Referer: https://stackoverflow.ai/" \
  -H "User-Agent: Mozilla/5.0" \
  -d '[[["Hello world"], "en", "zh-CN"], "te_lib"]'
```

---

## 5. 响应体

同样是嵌套数组，**翻译结果带 HTML 标签**：

```json
[
  ["<翻译结果 HTML>"],
  null,
  ...
]
```

取值路径：`response[0][0]` → 翻译后的 HTML 字符串。

### HTML 清洗

上游返回的不是纯文本，是片段 HTML（带 `<i>`、`<b>` 等）。清洗流程：

1. `BeautifulSoup(html, "html.parser").get_text()` → 纯文本
2. 去零宽空格 `​`
3. `markdownify` → Markdown（可选）

参考 `googletranslate_provider.py:_clean_response`。

---

## 6. 错误格式

非 200 时返回数组格式错误：

```json
[<code>, "<message>", [["<type>", ...]]]
```

- `[1]` 是可读 message
- `[0]` 是错误码

参考 `_parse_upstream_error`。

常见错误：

| 场景 | 表现 |
|------|------|
| Key 失效 | 403 / 401 |
| 频率限制 | 429 |
| payload 格式错 | 400 |

## 7. 直连最小 Python 示例

```python
import httpx

def translate(text: str, src: str, tgt: str, key: str) -> str:
    headers = {
        "Content-Type": "application/json+protobuf",
        "x-goog-api-key": key,
        "Origin": "https://stackoverflow.ai",
        "Referer": "https://stackoverflow.ai/",
        "User-Agent": "Mozilla/5.0",
    }
    payload = [[[text], src, tgt], "te_lib"]
    r = httpx.post(
        "https://translate-pa.googleapis.com/v1/translateHtml",
        headers=headers, json=payload, timeout=60,
    )
    r.raise_for_status()
    return r.json()[0][0]

if __name__ == "__main__":
    print(translate("Hello world", "auto", "zh-CN", "YOUR_KEY"))
```

## 8. 对接其他客户端

上游**不兼容 OpenAI 格式**。要让 OpenAI 客户端（Cherry Studio、NextChat、OpenWebUI 等）用上，必须保留本项目代理层做转换：

| 客户端格式 | 上游格式 |
|-----------|---------|
| `POST /v1/chat/completions` | `POST /v1/translateHtml` |
| `Authorization: Bearer <token>` | `x-goog-api-key: <KEY>` |
| `{"messages":[{"content":"..."}]}` | `[[["..."],"src","tgt"],"te_lib"]` |
| `{"choices":[{"message":{"content":"..."}}]}` | `[["<html>"]]` |
| SSE 流式 `data: {...}` | 非流式，整段返回 |

**直连只适合**：自写脚本、CLI 工具、不需要 OpenAI 兼容的后端服务。

**生产建议**：保留本项目代理，客户端打 `http://你的服务:8088/v1/chat/completions`，由代理转发上游并隐藏 Key。

---

## 9. 限制

- **非流式**：上游一次返回整段，无 token 级流式
- **Key 易失效**：高频或长期使用会被 Google 吊销
- **无官方文档**：此端点未公开，参数随时可能变
- **频率限制**：未知具体阈值，建议加缓存 + 限流
